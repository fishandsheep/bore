# Product

## Register

product

## Users
个人开发者在本机开发、联调或自托管维护过程中，使用这个本地 Web 控制台来管理 `bore local` TCP 隧道。使用场景通常发生在浏览器与终端并行打开的桌面环境里，用户希望用最少心智负担查看状态、创建配置、启动或停止隧道，并快速判断当前转发是否正常。

## Product Purpose
这个产品为 `bore --web` 提供一个本地控制平面，让用户不必只依赖命令行参数记忆和重复输入，也能安全、快速地管理本地 TCP 隧道。成功标准是：常见隧道操作能在数秒内完成，状态与日志一眼可读，界面在高频使用下保持克制、稳定、低干扰。

## Brand Personality
品牌气质应是：克制可靠、基础设施级、简洁高效。语气应冷静直接，优先传达可预期性、可操作性和技术信任感，而不是营销感、装饰感或情绪渲染。界面应让用户感觉它像一块稳定的本地控制面板，而不是展示型网站。

## Anti-references
不要像营销站：避免夸张 hero 结构、宣传式文案、过度品牌叙事和视觉噱头。不要太玻璃拟态：避免依赖模糊、悬浮半透明层和装饰性光效来制造“高级感”。不要像花哨低代码面板：避免过多彩色模块、过度圆角、冗余状态徽章、视觉密度失控和不必要的组件花样。

## Design Principles
1. 关键状态先于装饰表达：用户必须先看到隧道是否运行、连到哪里、最近发生了什么。
2. 把高频操作压到最短路径：创建、启动、停止、刷新、删除都应直接、可预测、少跳转。
3. 保持基础设施工具的信任感：用一致、稳定、低惊扰的组件语言，而不是追逐新奇视觉。
4. 浏览器表面要与命令行心智兼容：字段、状态、术语和行为应与 `bore` CLI 保持连续性。
5. 默认照顾可读性与可操作性：即使不是严格合规型内部工具，也要保证基本键盘操作、足够对比度和减少动画干扰。

## Accessibility & Inclusion
目标不是正式的高强度合规流程，但应尽量可用。至少保证清晰文本对比、表单标签明确、状态信息可辨、基础键盘操作可完成，并为偏好减少动画的用户提供低运动干扰体验。由于主要场景是本机开发时使用，界面应优先服务长时间桌面阅读与快速扫视。
